What is SSL?

Introduction

It is now somewhat of a conversation starter to mention the internet and how there is so much dependency on corporations and governments to protect their users' data. So much so that it is pretty much an annual event to write about how the White House or Pentagon experienced a cyber breach. This write-up will discuss certain aspects of cyber security that it takes to ensure your website is up to date.

HTTP

One of the most easily spotted traits of cyber security sophistication is HTTP or HTTPS; while my dad would call those the same, there is quite a significant distinction between the two. For starters, both can be vaguely described as a protocol that is the foundation for data transmission across the World Wide Web. HTTP (HyperText Transfer Protocol) is a set of instructions that operate asynchronously via a request to response. First, the client will send a request, to which the server replies with a response. All information sent is formatted in A text; these steps are HTTP transactions.

HTTPS

In contrast with HTTP, Hypertext Transfer Protocol Secure (HTTPS) is a newer, more secure way of transmitting data across the internet. HTTPS can easily be identified on websites by checking if there is a padlock next to the URL. Being equipped with TLS or SSL, you can think of it like that one younger sibling that consistently gets better grades than you. HTTP requests are broken up into packets while still readable text and shipped off. Problems arise when sniffers can access and read these packets using widely available software. On the contrary, HTTPS will encrypt your packets from plain text into meaningless characters, hiding away any passwords, bank details, or other information transmitted.

Encryption

Just like you cannot have chemistry or physics without mathematics being the underlying backbone, no web security can exist without encryption. Encryption is how we take a readable set of text, usually private, and change it to nonsensical characters. The only way to decrypt it would be by knowing the key, which would only be shared with the authorized parties. We encounter issues when trying to do this over the internet as it is risky to share this via email in the case of middle-men intrusion. Thus we created what is called asymmetric encryption.


SSL & TLS

Now that we know all our prerequisites, we must connect the dots to form the encryption-based security protocol known as Secure Sockets Layer (SSL). SSL is integrated into HTTPS that uses asymmetric encryption to encrypt all transferred information, making so any data obtained by the middle-men will be a collection of scrambled characters. SSL had been improved several times to keep up with more advanced attacks and later became the predecessor of a new security protocol, known as TLS.
